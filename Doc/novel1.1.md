### 성장과정 및 개발자가 되기까지

저는 어릴 때부터 구조화된 사고를 요구하는 퍼즐과 수학 문제를 좋아했습니다. 루빅스 큐브를 맞추기 위해 공식을 외우는 데 그치지 않고, 왜 그런 공식을 쓰는지 스스로 증명해 보는 과정에서 논리적 사고의 즐거움을 느꼈고, 자연스럽게 수학과에 진학했습니다. 대학 시절 알파고와 이세돌의 대결을 계기로 인공지능과 프로그래밍에 매료되었고, 컴퓨터 동아리 활동을 통해 C++로 문제를 해결하는 경험을 쌓으면서 “코드를 통해 세상의 문제를 푸는 일”을 제 진로로 정하게 되었습니다.

프론트엔드를 선택한 계기는 제가 좋아하던 게임 데이터를 더 잘 보여주고 싶다는 단순한 호기심에서 시작됐습니다. 사용자가 원하는 정보를 직관적으로, 빠르게 전달하는 화면을 직접 만들고 싶어 JavaScript와 React를 본격적으로 공부하기 시작했고, 멀티캠퍼스 프론트엔드 개발자 과정(React)에서 팀 프로젝트를 통해 실전 경험을 쌓았습니다. 이때의 경험이 지금도 새로운 기술을 배울 때마다 “실제 사용자에게 어떤 가치를 줄 수 있을까?”를 우선해서 생각하는 저만의 기준이 되었습니다.

### 업무 및 프로젝트 경험

현재 저는 프론트엔드 개발자로 약 2년 간 웹 서비스 개발과 운영을 경험해 왔습니다. 주로 React를 사용한 SPA 및 Next.js 기반의 SSR/SSG 프로젝트에 참여해 왔으며, 기획·디자인·백엔드와 긴밀히 협업하여 사용자 경험을 개선하는 데에 집중해 왔습니다.

1. **내부/외부 사용자용 웹 서비스 개발 경험**

- React, TypeScript, Zustand(Recoil) 등을 활용해 데이터 중심의 화면을 설계하고 구현했습니다.
- REST API 및 GraphQL API와 연동하며, API 스키마 변경이나 응답 형식 변경에 유연하게 대응할 수 있도록 타입 정의와 API 모듈을 분리해 관리했습니다.
- 공통 UI 컴포넌트와 훅을 추출해 재사용성을 높였고, 이를 통해 신규 페이지 개발 속도를 단축하고 유지보수성을 향상시켰습니다.

2. **사이드 프로젝트 ALOA – 게임 유저 정보 조회 및 매칭 서비스**

멀티캠퍼스 교육 수료 후, 수업에서 배운 내용을 실제 서비스로 연결해 보고 싶어 팀을 모아 ALOA 프로젝트를 진행했습니다. 스마일게이트에서 제공하는 게임 API를 사용해 사용자가 자신의 게임 캐릭터 정보를 한눈에 조회하고, 이를 기반으로 파티를 구성하거나 함께 플레이할 사람을 찾을 수 있는 서비스를 기획·개발했습니다.

- **기술 스택**: React, TypeScript, Styled-components(or TailwindCSS), Node.js/Express, AWS(EC2, S3), Nginx
- **주요 역할**
  - 전체 UI 설계 및 핵심 화면(캐릭터 리스트, 상세 스탯, 파티 매칭 화면) 개발
  - 공통 레이아웃, 카드 컴포넌트, 필터/정렬 컴포넌트 등 재사용 가능한 컴포넌트 설계
  - API 응답 구조에 맞춘 데이터 매핑 로직과 에러/로딩 상태 관리 구현
- **문제 상황 및 해결**
  - 예상보다 훨씬 많은 동시 접속자가 몰리며 서버가 다운되는 문제가 발생했습니다.
  - 서버 로그와 브라우저 네트워크 탭을 함께 분석해 과도한 API 호출과 비효율적인 렌더링 패턴을 발견했고,
    - 프론트에서는 요청 디바운싱, 캐싱, 불필요한 리렌더링 최소화(컴포넌트 메모이제이션, 훅 분리 등)
    - 서버에서는 캐시 정책 보완 및 스케일 업/아웃 전략 검토
      를 적용했습니다.
  - 이를 통해 동시 접속자가 초기 예상치보다 50% 이상 증가해도 안정적으로 서비스가 유지될 수 있게 개선했습니다.
- **운영 및 고도화**
  - 관리자 페이지를 별도로 구축해 사용자 통계 조회, 신고 관리, 공지 등록 등 운영 기능을 제공했습니다.
  - 반복적인 운영 업무를 줄이기 위해 간단한 유틸리티 기능(예: 일괄 처리, 상태 필터링)을 추가하면서, 실제 운영 환경에서 요구되는 “작지만 중요한 기능들”에 대한 감각을 기를 수 있었습니다.
  - 서비스는 입소문을 통해 유입이 꾸준히 유지되었고, 사용자 피드백을 바탕으로 UI 개선과 기능 보완을 계속해 나가며 “배포 후의 개발”의 중요성을 체감했습니다.

3. **팀 프로젝트 리딩 경험**

교육 과정에서 진행한 위치 기반 축제 알림 서비스 프로젝트에서는 팀장을 맡아 다양한 전공의 팀원들과 협업했습니다. 각자의 목표와 역량이 달라 일정 관리와 진행률 파악에 어려움이 있었지만, 매일 두 차례 스탠드업 미팅을 제안해 진행 상황을 공유하고, 우선순위 기반으로 작업을 재분배하는 방식으로 팀의 속도를 맞췄습니다. 그 결과, 네이버 SMS API와 카카오맵 API를 안정적으로 연동해 기한 내에 서비스를 완성했고, 프로젝트는 최우수상을 수상했습니다. 이 경험은 이후 실무에서 스스로 업무를 정리하고, 필요한 조율을 먼저 제안하는 데 큰 도움이 되었습니다.

### 강점과 보완점

저의 가장 큰 강점은 **새로운 기술을 두려워하지 않고 곧바로 작은 실험부터 해 보는 실행력**입니다. 새로운 라이브러리나 도구를 접하면 문서만 읽고 넘어가는 것이 아니라, 작은 프로토타입을 만들어 직접 적용해 보며 이해도를 높입니다. 예를 들어, 최근에는 UI 컴포넌트 개발 효율을 높이기 위해 `스토리북`을 도입해 보았습니다. 기존 프로젝트 일부를 스토리북 기반으로 분리해 컴포넌트 단위 개발·테스트 환경을 구축했고, 디자이너와의 커뮤니케이션에도 도움이 되는 것을 확인한 뒤 팀 내에 공유한 경험이 있습니다.

또 하나의 강점은 **집중력과 끈기**입니다. ALOA 프로젝트 당시에는 회사 업무 이후에도 시간을 쪼개 거의 하루 15시간에 가까운 시간을 서비스 개선에 투자한 적이 있습니다. 새로운 기능을 빠르게 도입하기보다는, 실제 사용자가 겪을 수 있는 문제를 줄이는 것에 우선순위를 두고 집중적으로 개선 작업을 반복했습니다.

다만, 저는 장기적인 커리어 플랜을 세우고 체계적으로 로드맵을 관리하는 부분에서는 아직 성장 중이라고 생각합니다. 지금까지는 “재미있어 보이는 것, 사용자가 좋아할 것 같은 것”을 기준으로 우선순위를 정해 달려온 편입니다. 앞으로는 회사의 비즈니스 방향성과 팀의 기술 전략 속에서 제가 어떤 역할을 맡아야 하는지 더 명확히 정의하고, 선배 개발자와 리더들의 피드백을 적극적으로 받아 중장기적인 성장 경로를 설계하고자 합니다.

### 입사 후 포부

저는 새로운 디스플레이와 인터랙션 기술이 만들어낼 사용자 경험에 큰 관심을 가지고 있습니다. 매년 CES를 챙겨 보며 플렉서블 디스플레이, 투명 디스플레이 등 차세대 디스플레이가 실제 제품으로 구현되는 과정을 지켜보면서, “이러한 하드웨어적 혁신 위에서 어떤 소프트웨어 경험을 제공할 수 있을까?”를 자주 상상해 왔습니다. 화면 전환이 자연스럽고, 사용자가 여러 디바이스와 화면 사이를 오가더라도 맥락을 잃지 않는 UX를 만드는 것이 제가 프론트엔드 개발자로서 추구하는 목표입니다.

입사 후에는 회사가 제공하는 서비스의 도메인과 사용자에 대한 이해를 바탕으로, **사용자가 직관적으로 이해할 수 있는 인터페이스와 일관된 경험**을 만드는 데 기여하고 싶습니다. 이를 위해 단순히 UI를 구현하는 데 그치지 않고, 디자인 시스템과 컴포넌트 아키텍처를 고민하며, 퍼포먼스 최적화와 접근성 향상까지 아우르는 프론트엔드 개발자로 성장하겠습니다.

중장기적으로는 새로운 디스플레이와 디바이스 환경에서도 자연스럽게 동작하는 웹/앱 경험을 설계하고 구현할 수 있는 개발자가 되고 싶습니다. 귀사의 제품과 서비스가 이러한 미래 환경에서도 경쟁력을 유지할 수 있도록, 저는 끊임없는 학습과 실험을 통해 프론트엔드 영역에서 믿고 맡길 수 있는 동료가 되겠습니다.
