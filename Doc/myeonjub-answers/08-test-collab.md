# 8. 테스트·협업

## 단위·통합·API 테스트, 테스트 피라미드

**답변 요지:**  
**단위**는 함수·클래스를 모킹과 함께 빠르게 많이. **통합**은 DB·메시지큐 등 실제 의존성 일부를 올려 검증. **API(E2E에 가까운)** 는 적게. 피라미드는 아래(단위)가 넓고 위로 갈수록 좁아집니다.

---

## TDD

**답변 요지:**  
TDD를 해 본 적이 있으면 **실패 테스트 → 구현 → 리팩터** 순으로 회귀를 줄였습니다. 적용 경험이 없으면 **원리는 이해하지만 프로젝트에서는 아직 쓰지 않았습니다.**
