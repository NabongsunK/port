# 1. 언어·런타임

## 동기·비동기 차이, 논블로킹 I/O

**질문:** 동기·비동기랑 논블로킹 I/O 차이를 설명해 달라.

**답변:**

동기면 호출한 작업이 끝날 때까지 호출자가 다음 줄로 안 넘어가고 기다립니다. 비동기면 호출 직후 다음 코드로 진행하고, 완료는 콜백·이벤트·Promise 같은 방식으로 나중에 받습니다.

논블로킹 I/O는 데이터가 준비될 때까지 스레드나 이벤트 루프가 꽉 막혀 있지 않고, 준비 상태를 보고 다른 작업을 이어가게 하는 식으로 **한 실행 단위가 여러 I/O를 번갈아 처리**할 수 있는 모델입니다. 서버에서 비동기 API를 쓰는 경우에도, 밑에는 보통 논블로킹 I/O나 스케줄러가 같이 깔려 있는 경우가 많습니다.

## `async`/`await`, `Promise`·콜백·에러 처리

`Promise`는 비동기 결과를 값처럼 다루기 위한 객체고, `then`/`catch`로 성공과 실패를 나눕니다. `async` 함수는 항상 Promise를 반환하고, `await`는 그 Promise가 끝날 때까지 **그 async 함수 안에서만** 실행 흐름이 멈춘 것처럼 보이게 해 줍니다.

에러는 `try`/`catch`로 잡거나 `.catch()`로 처리하고, reject된 Promise를 그냥 두지 않는 게 중요합니다. 재시도가 필요하면 횟수와 간격에 상한을 두고, 지수 백오프 같은 정책을 쓰는 편이 안전합니다.

## Node.js: 이벤트 루프, 싱글 스레드의 의미·한계

JavaScript 코드 실행은 한 스레드에서 이루어지는 게 일반적이고, 파일·네트워크 I/O는 libuv가 스레드 풀 등으로 밑에서 처리합니다. 그래서 I/O 위주 서버에는 잘 맞는 편이고요. 반대로 CPU를 오래 쓰는 작업을 그 스레드에서 돌리면 이벤트 루프가 막혀서 다른 요청까지 전체가 느려질 수 있습니다. 그런 부분은 워커 스레드로 빼거나 별도 서비스로 분리하는 식으로 대응합니다.

## 스레드 vs 프로세스, 동시성·병렬성 (멀티스레드 언어)

프로세스는 메모리·핸들 같은 자원을 따로 갖고, 스레드는 같은 프로세스 안에서 메모리를 공유합니다. 동시성은 여러 작업이 번갈아 진행되는 것이고, 병렬성은 실제로 여러 코어에서 동시에 실행되는 쪽에 가깝습니다. 멀티스레드에서는 공유 자원에 동시에 접근하면 레이스가 나기 쉬워서 락이나 원자적 연산 등으로 막아야 합니다.

## 예외 처리·재시도·타임아웃

외부 API나 DB 호출에는 타임아웃을 걸어서 무한 대기는 피하고, 5xx나 일시적 네트워크 오류처럼 “다시 시도해 볼 만한” 경우에만 재시도하되 횟수와 간격에 한도를 둡니다. 4xx 같은 비즈니스 오류는 같은 요청을 반복해도 의미가 없는 경우가 많아서 재시도하지 않는 편이 맞습니다. 로그에는 요청 ID나 상대 서비스 이름을 남겨서 나중에 추적할 수 있게 합니다.
