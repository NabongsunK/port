# 7. 인프라·DevOps·도구

## Git — 브랜치, merge vs rebase, 충돌

**답변 요지:**  
기능 브랜치·PR로 리뷰 후 메인에 합칩니다. **merge**는 히스토리 보존, **rebase**는 선형 히스토리 — 팀 규칙에 맞춥니다. 충돌은 같은 파일의 의도를 맞춰 수동 해결합니다.

---

## Docker — 이미지·컨테이너·멀티 스테이지

**답변 요지:**  
이미지는 실행 환경을 고정하고, 컨테이너는 격리된 프로세스입니다. **멀티 스테이지 빌드**로 빌드 도구는 빼고 런타임만 얇은 이미지에 넣습니다.

---

## CI/CD

**답변 요지:**  
푸시/PR 시 **빌드·린트·테스트**를 자동으로 돌리고, 메인 머지 시 **스테이징/프로덕션 배포**까지 파이프라인으로 연결합니다(본인 경험에 맞게 구체화).

---

## 리버스 프록시·로드 밸런싱

**답변 요지:**  
Nginx가 TLS 종료·정적 파일·`proxy_pass`로 앱으로 넘깁니다. 로드 밸런서는 여러 인스턴스에 트래픽을 나눕니다. **헬스체크**로 죽은 노드를 빼는 맥락을 짚습니다.

---

## 헬스체크·그레이스풀 셧다운

**답변 요지:**  
`/health`로 준비 상태를 알리고, 배포 시 **새 연결 거부 후 진행 중 요청 마무리**하고 프로세스를 끕니다(K8s preStop·SIGTERM 등 경험이 있으면 연결).

---

## 모니터링·APM

**답변 요지:**  
메트릭(요청 수, 지연, 에러율)·로그·분산 추적으로 장애를 좁힙니다. 사용 도구가 있으면 이름을 대고, 없으면 개념만으로도 됩니다.
