# 면접 예상 질문 — 2년차 백엔드 위주

> 프론트(브라우저·CSS·React 등) 위주 항목은 제외하고, 서버·DB·API·운영 쪽으로 정리했습니다.

---

## 1. 언어·런타임 (Node / Java / Python 등 본인 스택 기준)

- 동기·비동기 차이, 논블로킹 I/O 개념
- `async` / `await`, `Promise`·콜백·에러 처리 패턴
- Node.js라면: 이벤트 루프, 싱글 스레드 모델의 의미·한계
- 멀티스레드 언어를 쓴다면: 스레드 vs 프로세스, 동시성·병렬성 차이 (개념 수준)
- 예외 처리·재시도·타임아웃 설계 경험

---

## 2. 네트워크·HTTP (서버 관점)

- 주소창에 URL 입력 후 서버까지 요청이 가는 흐름 (DNS, TCP, TLS)
- HTTP vs HTTPS, 상태 코드(4xx·5xx) 의미와 대응
- HTTP 메서드(GET/POST/PUT/PATCH/DELETE)와 멱등성
- 헤더: `Content-Type`, 인증 관련, `Cache-Control` 등
- CORS가 왜 생기는지, 서버에서 어떻게 처리하는지
- 쿠키·세션·토큰을 서버에서 어떻게 검증·발급하는지

---

## 3. API 설계·보안

- RESTful API 설계 원칙, 리소스 네이밍·버전 관리
- REST vs GraphQL vs gRPC — 각각 언제 쓰기 좋은지(개념)
- 인증(Authentication)과 인가(Authorization) 차이
- JWT 구조·만료·갱신·저장 시 주의점
- CSRF, XSS, SQL Injection 방어 — 서버에서 할 수 있는 것
- 비밀번호 저장(해시·솔트), 환경변수·시크릿 관리

---

## 4. 데이터베이스·SQL (핵심)

### 4.1 기본

- RDBMS와 NoSQL 차이, 각각 언제 선택하는지
- 정규화(1~3정도)와 반정규화 트레이드오프
- PK·FK·제약조건, CASCADE 의미

### 4.2 쿼리·성능

- `JOIN` 종류(INNER, LEFT, …)와 실행 계획을 생각하는 습관
- 인덱스란 무엇인지, B-Tree 개념, 과도한 인덱스의 부작용
- 커버링 인덱스, 복합 인덱스 컬럼 순서
- N+1 문제란 무엇인지, 해결 방법(조인·배치·DataLoader 등)
- 풀 스캔 vs 인덱스 스캔, `EXPLAIN`(또는 유사 도구) 경험

### 4.3 트랜잭션

- ACID, 트랜잭션 경계를 어디에 두는지
- 격리 수준(Read Committed, Repeatable Read 등)과 팬텀·더티 리드 개념
- 락(Lock) — 낙관적/비관적, 데드락이 나는 상황과 회피

### 4.4 운영·아키텍처

- 커넥션 풀링이 왜 필요한지
- 마이그레이션 도구(Flyway, Liquibase 등) 경험 여부
- 샤딩·파티셔닝·리플리케이션 개념(업무 깊이에 맞게)
- 사용 DB(MySQL, PostgreSQL, MSSQL, Oracle 등) 특징 한두 가지

---

## 5. 캐시·비동기 처리

- 인메모리 캐시(Redis 등) 사용 경험 — 키 설계, TTL, 캐시 무효화
- 세션 스토어·분산 락으로 Redis를 쓴 경험(있다면)
- 메시지 큐(Kafka, RabbitMQ, SQS 등) — 왜 쓰는지, 최소 한 번은 설명할 수 있게
- 웹소켓 vs SSE vs 폴링 — 실시간 알림·로그 스트리밍 맥락

---

## 6. 백엔드 아키텍처·품질

- 레이어드 아키텍처(Controller–Service–Repository 등) 경험
- MVC, DDD 일부 개념(애그리거트·도메인 서비스 수준은 선택)
- SOLID·디자인 패턴(싱글톤, 팩토리, 전략 등) 실무 예
- API 버저닝, 하위 호환, Deprecation 처리
- 멱등 키·중복 요청 방지(결제·주문 등)
- 오프셋 vs 커서 기반 페이지네이션
- 로깅(구조화 로그, 요청 ID), 에러 응답 포맷 통일

---

## 7. 인프라·DevOps·도구

- Git — 브랜치 전략, `merge` vs `rebase`, 충돌 해결
- Docker — 이미지·컨테이너, 멀티 스테이지 빌드 개념
- CI/CD 파이프라인 경험(빌드·테스트·배포)
- 리버스 프록시(Nginx)·로드 밸런싱 개념
- 헬스체크·그레이스풀 셧다운
- 모니터링·APM(개념 또는 사용 도구)

## 8. 테스트·협업

- 단위·통합·API 테스트 경험, 테스트 피라미드 개념
- TDD 경험 여부
- 코드 리뷰 시 백엔드에서 중요하게 보는 것

- 가장 최근에 해결한 기술 문제(장애·성능·설계 포함)
- DB 병목·쿼리 튜닝·트랜잭션 이슈를 다뤄본 경험
- 사용한 프레임워크(Spring, Express, FastAPI 등)를 선택한 이유
- 트래픽·데이터량 증가에 맞춰 구조를 바꾼 경험
- 레거시 API·스키마와의 호환을 유지한 경험
- 성능 이슈를 어떻게 재현·측정했는지

## 10. 인성·지원 동기·일반

- 자기소개, 강점·약점
- 지원 동기, 어떤 개발자가 되고 싶은지
- 입사 후 1년 목표·성장 계획
- 일정이 촉박할 때 우선순위 정하는 방법
- 기획·프론트·인프라와 협업할 때 경험
- 실패·실수에서 배운 점
- 스타트업 **하고 싶은 일 / 해야 할 일 / 할 수 있는 일**(해당 시)
- 지원 부서 외 배치 시, 대학·취미·독서 등 기존 항목
