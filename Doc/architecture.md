# port — 구조·설계 메모

포트폴리오/블로그용 Next.js 앱(`port`)의 **기술 구조**와, 자소서·면접에서 **역할을 어떻게 쓸지** 정리한 문서입니다.

---

## 1. 한 줄 요약

- **Next.js 12** 기반 정적·SSR 혼합 페이지, **Contentlayer**로 MDX/마크다운 콘텐츠를 타입 안전하게 빌드.
- **`basePath: "/port"`** 로 서브 경로(`/port`)에 배포되도록 설계.
- 프로덕션은 **3030 포트** + **PM2**로 상시 실행, `script/auto_sync.sh`로 원격 변경 감지 후 빌드·재시작(리눅스/맥 서버 전제).

---

## 2. 레이어 구조 (개념)

| 구역                     | 역할                                                |
| ------------------------ | --------------------------------------------------- |
| `app/`                   | App Router 진입 페이지, 전역 스타일 등              |
| `pages/`                 | 페이지 라우트(기존 Pages Router와 병행 가능한 구성) |
| `posts/`                 | MDX 등 콘텐츠 소스(Contentlayer 입력)               |
| `components/`            | UI 블록(About, Blog, 공통 레이아웃 등)              |
| `contentlayer.config.ts` | 콘텐츠 스키마·빌드 파이프라인 정의                  |
| `utils/`                 | 공용 유틸                                           |

콘텐츠는 **파일 기반** → 빌드 시 **Contentlayer**가 생성 타입·메타데이터를 만들고, **Next**가 페이지로 노출하는 흐름입니다.

---

## 3. 배포·URL 경계

- 브라우저 기준 경로: **`/port` 하위** (예: `https://도메인/port`).
- 로컬 개발: `npm run dev` → `next dev -p 3030` (README 기준).
- 프로덕션: `next start -p 3030`, PM2로 프로세스 관리.

역프록시(Nginx 등)에서 `/port`를 이 앱으로 넘기는 경우, **앱의 `basePath`와 서버 프록시 규칙이 일치**해야 합니다.

---

## 4. 다른 프로젝트(예: wwmw)와 “한 화면에 묶어서” 보여줄 때

- **의미 있게 합치는 것**은 좋습니다. 다만 문서/포트폴리오에 **각 조각의 책임**을 한 줄이라도 적어 두는 것이 좋습니다.
- 예: “진입·포트폴리오 UI는 `port`, ○○ 처리는 `wwmw` …”처럼 **경계**를 드러내면 설계 이해도가 올라갑니다.

---

## 5. 자소서·면접에서 쓰는 원칙 (역할)

- **전체 아키텍처를 “내가 다 설계했다”고 쓰지 않는다.** 팀·리드 설계가 있으면 그렇게 밝힌다.
- **본인이 한 것**을 쓴다: 구현 범위, 모듈 내부 구조, 에러/성능/테스트 판단, Contentlayer·라우팅·배포 연동 등 **결정에 참여한 부분**.
- **작은 설계도 설계**로 쓸 수 있다: 컴포넌트 분리, 콘텐츠 파이프라인, `basePath`/배포 스크립트와의 맞춤 등.

이 문서는 위 원칙에 맞춰 **사실만** 정리하고, 수치·성과는 본인이 측정한 값으로 채우면 됩니다.
